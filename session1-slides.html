🤔 Why Git?
연구자에게 Git이 필요한 이유
😱 Git 없을 때
analysis_final.py
analysis_final2.py
analysis_final_진짜최종.py
analysis_20241103.py
analysis_제출용.py
✅ Git 있을 때
analysis.py
# 모든 변경 이력이 자동 기록
# 언제든 이전 버전으로 복원
# 지도교수와 실시간 협업
Git = 논문 작업의 "타임머신" + "협업 도구"
코드·데이터·문서의 모든 변경 이력을 저장하고, 실수해도 언제든 되돌릴 수 있습니다.

🛠 실습 1 계속
저장소를 내 컴퓨터로 가져오기 clone
GitHub에 만든 저장소를 내 컴퓨터에 복사하는 과정을 Clone(클론) 이라고 합니다.
bash 
# 원하는 폴더로 이동 (예: 바탕화면)
cd ~/Desktop

# 저장소 클론 (URL은 GitHub 페이지의 초록색 Code 버튼에서 복사)
git clone https://github.com/사용자명/finedust-research.git

# 클론된 폴더로 진입
cd finedust-research

# 내용 확인
ls
💡 팁
GitHub 저장소 페이지 → 초록색 <> Code 버튼 → HTTPS 탭에서 URL을 복사하세요.

⌨️ 실습 2
핵심 명령어 add · commit · push
bash — 파일 수정 후 GitHub에 올리는 전체 흐름 
# 1. 현재 상태 확인 (항상 먼저!)
git status

# 2. 변경된 파일을 Staging에 추가
git add analysis.py           # 특정 파일만
git add .                     # 변경된 모든 파일

# 3. 메시지와 함께 커밋 (스냅샷 저장)
git commit -m "PM2.5 데이터 전처리 코드 추가"

# 4. GitHub에 업로드
git push origin main

# 5. GitHub에서 최신 내용 가져오기
git pull origin main
💡 좋은 커밋 메시지 규칙
"무엇을 왜 했는지" 를 한 줄로 — 예: "서울 2023년 PM2.5 이상치 제거 로직 수정"

📋 치트시트
자주 쓰는 Git 명령어
명령어 설명 언제 사용?
git status 현재 변경 사항 확인 작업 시작할 때마다
git log --oneline 커밋 이력 간단히 보기 이전에 뭘 했는지 확인할 때
git diff 수정 내용 비교 커밋 전 변경사항 검토
git restore 파일명 수정 취소 (되돌리기) 실수로 파일 망쳤을 때
git clone URL 원격 저장소 복사 처음 프로젝트 받아올 때
git pull 원격 변경사항 가져오기 작업 시작 전 동기화

*.csv
*.nc
*.h5

# Python 캐시
__pycache__/
*.pyc
.ipynb_checkpoints/

# 환경설정 & API 키 (절대 업로드 금지!)
.env
config_secret.py
api_keys.txt

.DS_Store          # Mac
Thumbs.db          # Windows
