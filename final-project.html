항목미흡기본우수배점
GitHub 저장소 구조 파일이 흩어져 있고 README만으로 내용을 파악하기 어렵다. 권장 폴더 구조와 README가 있으며 주요 산출물 위치가 보인다. 커밋 기록, 폴더 구조, README 실행 안내가 일관되고 재현 가능하다. 15
레퍼런스 수집 품질 논문 수가 부족하거나 주제와 직접 연결되지 않는다. 논문 10편 목록과 A/B/C 스크리닝 이유가 정리되어 있다. 핵심 논문, 배경 논문, 제외 논문의 판단 기준이 명확하고 검색식이 남아 있다. 20
논문 요약 카드 요약이 짧거나 데이터, 방법, 결과 확인 항목이 빠져 있다. 요약 카드 5개 이상에 연구문제, 데이터, 방법, 결과, 한계가 채워져 있다. 각 카드가 원문 확인 항목과 내 논문에서의 사용 위치까지 연결된다. 20
Research Gap 명확성 Gap이 너무 넓거나 레퍼런스 근거 없이 주장된다. Reference Matrix를 바탕으로 기존 연구의 공통점과 부족한 점을 설명한다. 주장-근거 연결이 선명하고 Introduction 4문단으로 바로 옮길 수 있다. 20
Figure/분석 결과 재현성 그림만 있고 데이터 출처나 생성 코드가 불명확하다. Figure 1개 이상과 생성 코드, 데이터 경로, requirements가 있다. README만 보고 Figure를 다시 만들 수 있고 논문 수치와 코드 결과가 일치한다. 15
최종 원고 초안 Introduction 구조가 흐릿하거나 인용 위치가 없다. Introduction 5문단 초안과 Overleaf PDF가 제출된다. 각 문단의 역할이 분명하고 Citation Needed 또는 BibTeX 인용이 적절히 표시된다. 10
