Workflow

GitHub → Codex → Figures/Tables → Overleaf

1

GitHub 저장소 준비

데이터·코드·문서 산출물을 한 프로젝트 폴더에서 버전 관리합니다.

2

Codex에게 산출물 단위로 지시

코드 파일만 요청하지 말고 figure, table, docs, README 업데이트까지 한 번에 요청합니다.

3

Overleaf로 업로드·조립

기본 경로는 파일 업로드입니다. GitHub 연동은 계정/요금제가 가능한 경우 선택합니다.

Folder Structure

논문 프로젝트 폴더 구조

pm25-paper-project/
├── data/ # sample_pm25.csv
├── src/ # analyze_pm25.py
├── figures/ # pm25_monthly_trend.png
├── tables/ # summary_stats.tex
├── docs/ # result_summary.md
└── paper/ # main.tex, references.bib
원칙: 원자료는 보존하고, 코드로 산출물을 재생성하며, 논문에는 산출물 경로를 명확히 연결합니다.
Reframe Codex

Codex는 코드 생성기가 아니라 논문 산출물 생성기

나쁜 요청
  • PM2.5 분석 코드 짜줘
  • 그래프 예쁘게 만들어줘
  • 결과 문단 써줘
좋은 요청
  • 입력 데이터와 실행 명령을 고정
  • 그림·표·요약문 파일명을 지정
  • README에 재현 절차를 남기게 지시
오늘은 prompts/codex_paper_pipeline_prompt.md를 붙여넣어 산출물 중심으로 작업합니다.

Assembly

main.tex, references.bib, figures/ 조립

\includegraphics[width=0.85\textwidth]{figures/pm25_monthly_trend.png}
\input{tables/summary_stats.tex}
\cite{who2021airquality}
\bibliographystyle{plain}
\bibliography{references}
템플릿은 templates/overleaf_main_template.tex를 사용합니다. 그림과 표 경로가 실제 업로드 위치와 같은지 확인하세요.
Compile

Figure/Table/Citation 넣고 컴파일

Figure 오류

파일명 대소문자와 폴더 위치를 확인합니다.

Table 오류

LaTeX 특수문자 %, _, &가 escape 되었는지 확인합니다.

Citation 오류

BibTeX key와 \cite{} key가 같은지 확인합니다.

PDF가 나오면 초안 완성입니다. 완벽한 논문이 아니라 검토 가능한 재현 초안이 목표입니다.
